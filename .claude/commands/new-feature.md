When `/new-feature` is invoked:

1. Remind the user that this project is spec-first.
2. Ask for the feature name if it was not included.
3. Create a new markdown spec in `/docs/specs/` from `/docs/specs/spec-template.md`.
4. Use a kebab-case filename derived from the feature name.
5. Do not implement the feature in the same step unless the user asks after reviewing the spec.
